# Music Manager v0.4.0

Music Manager v0.4.0 adds explicit opt-in MusicBrainz candidate matching to the
existing read-only, report-driven scan and analysis workflow.

## Highlights

- Match album groups against MusicBrainz release groups and local recordings
  against MusicBrainz recordings.
- Review deterministic candidate evidence, confidence scores, margins, and
  matched, ambiguous, unmatched, not-eligible, or error outcomes.
- Reuse a private persistent cache with fixed-origin HTTPS requests,
  identifiable application requests, conservative rate limiting, bounded
  retries, and circuit-open behavior.
- Register the complete matching report family atomically as schema 1.1 while
  retaining compatibility with schema 1.0 scan runs.
- Continue to test all MusicBrainz behavior offline with injected clients,
  transports, clocks, sleepers, and synthetic responses.

## Explicit opt-in workflow

MusicBrainz is disabled by default. Match one finalized schema 1 run with:

```bash
music-manager match \
  --scan-run reports/<scan-id> \
  --musicbrainz
```

A local configuration can provide persistent consent for the `match` command:

```yaml
musicbrainz:
  enabled: true
```

The `--no-musicbrainz` flag overrides enabled configuration for one invocation.
Scan and analysis commands do not instantiate or call the MusicBrainz client.

## Privacy and safety

MusicBrainz may receive normalized artist, album, and title text. MusicBrainz
and the network operator can also observe the source IP, request timing, and
the application User-Agent.

Paths, filenames, scan IDs, file-record IDs, audio, artwork, and source files
are not sent. Matching does not open source-library paths stored in reports.

Matching writes reports only. It does not apply metadata, and no
source-library file is renamed, moved, copied, deleted, retagged, staged, or
edited. Version 0.4.0 adds no artwork lookup, acoustic fingerprinting,
background refresh, or polling.

## Generated reports

A completed match atomically registers these private reports in the selected
scan directory:

- `musicbrainz_album_groups.csv`
- `musicbrainz_album_candidates.csv`
- `musicbrainz_recording_candidates.csv`
- `musicbrainz_match_results.csv`

The reports contain reviewable evidence only. A `matched` status is not user
confirmation and does not authorize any library change.

## Compatibility

Music Manager v0.4.0 supports CPython 3.11 through 3.14. Matching accepts
complete or incomplete schema 1 scan runs, upgrades their manifest to schema
1.1 when registering matching reports, and does not rewrite primary scan
artifacts. Legacy unversioned v0.2 reports remain analysis-only and cannot be
matched.
